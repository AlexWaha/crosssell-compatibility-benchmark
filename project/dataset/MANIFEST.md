# AVTC Dataset Backup Manifest

Full backup of the AVTC dataset taken before removing OpenCart and restructuring
into `frontend/` + `backend/`. This is the source of truth for restoring data.

## Contents

| Path | What | Size |
|------|------|------|
| `sql/opencart.sql` | Full `opencart` database dump (products, descriptions, attributes, options, filters, categories, manufacturers, `oc_avtc_product_ai`, jobs) | 54M |
| `sql/avtc_metrics.sql` | Full `avtc_metrics` database dump (schema + data) | 12K |
| `typesense/` | Typesense data volume snapshot (collection `products`, embedding `num_dim:1024`) | ~668M |
| `images/` | Product images `catalog/product/**` | 1.6G |
| `json/` | Readable export: `products.json`, `categories.json`, `attributes.json` | ~18M |

## IMPORTANT NOTE

`recommendations`, `compatibility_evaluations`, `quality_snapshots` are **EMPTY**.
The compatibility (recommendation) pipeline has not been run yet. Products are
imported and normalized (`oc_avtc_product_ai` = 14767) and embeddings are indexed
in Typesense, but cross-sell recommendations still need to be generated by running
the compatibility engine in the new `backend/` (this incurs OpenAI cost).

## Restore notes

- MySQL: `mysql -uroot -p < sql/opencart.sql` then `< sql/avtc_metrics.sql` (dumps use `CREATE DATABASE`).
- Typesense: copy `typesense/` into the new Typesense data volume.
- Images: serve `images/` via the backend image route / nginx.
- DB name stays `opencart` so existing pipeline configs port over unchanged.
